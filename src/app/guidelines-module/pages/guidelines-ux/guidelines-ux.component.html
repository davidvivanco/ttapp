Enlace para Ir a "modularizar"

delete Borrar elementos / Dar de baja

Cada vez que se haga click en borrar o dar de baja se lanzará la modal de confirmación correspondiente. Explicando al usuario las consecuencias de la acción (en el caso de borrar no se puede deshacer).

notifications Notificaciones / Toast

Algunas acciones generan notificaciones o toasts en la aplicación.

Notificaciones

Las notificaciones se producen cuando un usuario recibe una acción/comunicación bien desde el back o desde otro usuario. Como por ejemplo cuando un admin actualice el cv de un usuario.
En el icono de la campana del header aparecerá el aviso de nueva notificación, al clikar en el icono verá una breve descripción de la notificación y un botón para verla. Si además el usuario está conectado en la aplicación visualizará también un toast indicándole lo sucedido.
Toasts

Los toasts se disparán cuando el usuario ha realizado alguna acción sobre la aplicación y recibe una respuesta de back (éxito, error, etc...). Por ejemplo, cuando actualiza sus datos personales o edita una ficha de puesto.

access_time Loaders

Para cargas de datos en las que el back puede tardar en dar respuesta utilizaremos el spinner / loader para indicar la espera. En caso de no recibir datos se añadirá un texto explicándolo.

Ejemplo: Resultados de búsqueda o tablas.

fingerprint Permisos y roles

Que un usuario pueda o no realizar alguna acción, o tener acceso a según que partes de esta, dependerá siempre de los permisos y rol que el usuario tenga.

Ejemplo: Un usuario sin permiso para editar fichas de puesto no debería tener esa opción en el menú de admin aunque tenga el rol de admin.

settingsConfiguración por empresa

No todas las instancias de Talentoo disponen de las mismas opciones o módulos. Siempre hay que desarrollar teniendo esto en cuenta para que al desplegar cada instancia se adapte al cliente que tenga por configuración.

speaker_notes Traducciones o textos por instancia

Al crear nuevos componentes o módulos deberemos tener en cuenta incluir los textos en los idiomas correspondientes (json de traducciones en back).
Si una instancia tuviera textos particulares se incluirían en los archivos de traducciones de dicha empresa.

view_module Modularizar la aplicación

Para evitar cargar toda la aplicación completa en todos los clientes tenemos que modularizar la misma y utilizar lazy loading. Así por configuración sabremos que módulos cargará o no el cliente.

Ejemplo: Fichajes, guía de estilos o CV.
