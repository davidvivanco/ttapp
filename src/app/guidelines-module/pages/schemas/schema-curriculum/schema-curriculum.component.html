El schema de CV se crea vacío al instanciar la aplicación.

El administrador puede ir añadiendo bloques al CV y elementos a estos bloques sin que el resto de usuarios lo vean.

Para que el resto de usuarios vean y puedan rellenar el CV el administrador deberá publicar el CV.

  • ¿Cómo funcionan blocks y versions en el objeto curriculum?
    • En blocks se guarda la version que ve el administrador en la edición.
    • En versions se van pusheando cada vez que el admin "publica" y la última de ese array es a la que los usuarios tendrán acceso para añadir datos.
    • La idea en un futuro es permitir al admin cambiar de version de CV por eso las vamos guardando.
