person Personal data

El schema de datos personales extra no se crea al instanciar la aplicación. Se crea la primera vez que el admin añade un input en la vista

El administrador puede ir añadiendo inputs sin que el resto de usuarios lo vean.

Para que el resto de usuarios vean y puedan rellenar los datos el administrador deberá publicar el schema.

  • ¿Cómo funcionan blocks y versions en el objeto personal data
    • En blocks se guarda la version que ve el administrador en la edición.
    • En versions se van pusheando cada vez que el admin "publica" y la última de ese array es a la que los usuarios tendrán acceso para añadir datos.
    • La idea en un futuro es permitir al admin cambiar de version por eso las vamos guardando.
